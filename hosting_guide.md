# Hosting Guide: Deploying to Render (Free Tier)

This guide walks you through hosting the **Tuition Centre Worksheet Generator** on [Render.com](https://render.com) for free. 

The service is configured to automatically run in **Webhook Mode** when hosted. Because Render's free tier puts services to sleep after 15 minutes of inactivity, webhooks allow Telegram to automatically wake the service up whenever a user clicks a button or sends a message!

---

## Prerequisites

1. A **GitHub** account containing your repository code.
2. A **Render** account (free to register at [render.com](https://render.com)).
3. Your **Supabase Project API URL & Key** (which you already set up).
4. Your **Gemini API Key** and **Telegram Bot Token**.

---

## Step-by-Step Deployment Instructions

### Step 1: Push Your Code to GitHub
Ensure all your project files (including the updated `pdfService.ts`, `server.ts`, and `bot/index.ts`) are committed and pushed to a private or public repository on GitHub.

### Step 2: Create a New Web Service on Render
1. Log in to the [Render Dashboard](https://dashboard.render.com).
2. Click **New +** and select **Web Service**.
3. Connect your GitHub account and select your `worksheet-generator` repository.

### Step 3: Configure Build & Start Settings
On the Web Service creation page, set the following parameters:

* **Name**: `worksheet-generator` (or any name you prefer)
* **Region**: Choose the region closest to you (e.g., Singapore or US Oregon)
* **Branch**: `main` (or your active development branch)
* **Language/Runtime**: `Node`
* **Build Command**: `npm install && npm run build`
* **Start Command**: `npm run start`
* **Instance Type**: Select the **Free** tier.

### Step 4: Add Environment Variables
Scroll down to the **Environment Variables** section and add the following keys:

| Key | Value | Description |
| :--- | :--- | :--- |
| `NODE_ENV` | `production` | Enables production optimisations. |
| `TELEGRAM_BOT_TOKEN` | `your_bot_token` | The token generated by `@BotFather` on Telegram. |
| `GEMINI_API_KEY` | `your_api_key` | Your Google Gemini API key. |
| `SUPABASE_URL` | `https://your-proj.supabase.co` | Your Supabase project URL. |
| `SUPABASE_KEY` | `your_anon_or_service_key` | Your Supabase database key. |
| `WEBHOOK_URL` | `https://worksheet-generator.onrender.com` | **CRITICAL**: The URL Render allocates for your web service (visible at the top of the Render project dashboard). Make sure it starts with `https://` and has no trailing slash. |

*Note: Once you input these, click **Create Web Service** at the bottom.*

## How the Sleep & Wake-up Cycle Works

### 1. Inactivity Sleep (15 Minutes)
Render's free tier monitors incoming HTTP requests. If no one accesses the Bulk Upload Web Dashboard and no messages are sent to the Telegram bot for 15 consecutive minutes, Render puts the server into a **suspended (sleep) state** to conserve resources.

### 2. Auto Wake-up on Bot Click / Upload
* When a user sends a message or presses an inline keyboard button in the Telegram bot, Telegram sends a secure HTTP POST request (`/telegraf/<secret-token>`) to your Render service.
* Render receives this request, detects incoming HTTP traffic, and **automatically resumes (wakes up)** the service.
* *Note: The very first interaction after a long period of inactivity may experience a delay (typically 30–50 seconds) while the container boots up. The bot will automatically reply as soon as the boot is complete.*
* Any subsequent interactions will respond instantly as long as the server remains awake.
* Accessing the web dashboard URL in your browser will also wake up the server in the exact same manner.
